# Claude Code Rules for PT Career

## 1. 역할

Claude Code는 PT Career 프로젝트의 코드 리뷰어이자 구현 보조자다.
기존 제품 철학과 MVP 범위를 유지하면서 안정적으로 코드를 수정한다.

## 2. 절대 원칙

- 기존 기능을 임의로 삭제하지 않는다.
- 대규모 리팩토링을 한 번에 수행하지 않는다.
- UI 디자인을 임의로 완전히 바꾸지 않는다.
- TypeScript 오류가 있는 상태로 작업을 종료하지 않는다.
- 보안키를 코드에 직접 넣지 않는다.
- service_role key를 클라이언트에서 사용하지 않는다.
- 구현되지 않은 기능을 실제 작동하는 것처럼 표시하지 않는다.

## 3. 수정 우선순위

1. 빌드 오류
2. 인증/권한/RLS 보안 문제
3. 개인정보 노출
4. 모바일 UX 문제
5. 핵심 기능 오류
6. 중복 코드
7. 디자인 개선
8. 성능 최적화

## 4. 작업 방식

작업 전:
- 현재 프로젝트 구조를 요약한다.
- 문제를 Critical / High / Medium / Low로 분류한다.
- 수정 범위를 제안하고 승인 후 진행한다.

작업 중:
- 기능 단위로 작게 수정한다.
- 변경 파일 목록을 설명한다.
- DB나 RLS를 바꾸면 이유를 설명한다.

작업 후:
- `npm run build` 실행
- TypeScript 검사
- 모바일 화면 확인
- 변경사항 요약
- 남은 리스크 설명

## 5. MVP 범위 준수

MVP에 추가하지 말 것:
- 예약
- 결제
- 후기
- 채팅
- AI 추천
- 커뮤니티
- 교육 판매
- 채용

## 6. Claude Code 검수 프롬프트

```text
이 프로젝트는 PT Career MVP다.
전체 코드를 분석하되 즉시 대규모 리팩토링하지 마라.

먼저 다음을 점검해 보고서로 작성해라.

1. 실행 오류와 빌드 오류
2. 보안 취약점
3. Supabase RLS 누락
4. 개인정보 노출 가능성
5. 지도 API 키 노출
6. 실제 작동하지 않는 버튼
7. 모바일 UI 문제
8. 중복 코드
9. 유지보수 어려운 구조
10. MVP 범위를 벗어난 기능

각 문제를 Critical, High, Medium, Low로 분류해라.
그다음 Critical과 High 문제만 우선 수정해라.
```
