## 13개 화면 상세 정의

### EXP-ONB-001: 로그인 후 최초 진입

**화면 ID**: EXP-ONB-001

**화면명**: 로그인 후 최초 진입

**사용자**: Google OAuth로 로그인한 신규 사용자 (프로필 없음)

**접근 조건**: `isAuthenticated === true` AND `profile === null`

**화면 목적**: 로그인한 사용자가 아직 전문가 프로필을 만들지 않았음을 이해하고, 프로필 작성 프로세스를 간단히 안내한다.

**진입 경로**: 
- Google OAuth 로그인 성공 후 자동 진입
- 소비자 홈 → 마이페이지 접근 시

**완료 조건**: 사용자가 "전문가 프로필 작성 시작" CTA를 클릭하여 EXP-ONB-002로 이동

**핵심 CTA**: `전문가 프로필 작성 시작` (Primary)

**보조 CTA**: `나중에 하기` (뒤로 가기)

**표시 데이터**:
- 환영 인사말
- 프로필 작성의 목적 (1줄 요약)
- 작성 단계 개요
- 자격 증빙이 필요한 이유
- 승인 전 비공개 안내
- 관리자 검토 후 공개됨 설명

**입력 데이터**: 해당 없음

**필수 항목**: 해당 없음

**선택 항목**: 해당 없음

**읽기 전용 항목**: 해당 없음

**사용자 변경 금지 항목**: 해당 없음

**상태별 UI**:
- **001 (로딩)**: 프로필 조회 중 Skeleton 표시
- **007 (권한 거부)**: 프로필이 있는 경우 정상 라우팅 (마이페이지 대시보드로 이동, 권한 거부 아님)
- **008 (세션 만료)**: "세션이 만료되었습니다. 다시 로그인해주세요."
- **009 (네트워크 오류)**: 프로필 조회 실패 → 재시도 CTA 제공

**오류 처리**: 
- 프로필 조회 실패: 재시도 및 홈으로 이동 옵션

**빈 상태**: 해당 없음 (고정 안내 화면)

**모바일 기준**:
- 360px에서 텍스트 줄바꿈 자연스러움
- CTA 최소 44px 높이
- 상단·하단 여백 충분

**접근성 기준**:
- 모든 텍스트 명도 충분 (대비 4.5:1 이상)
- 포커스 상태 명시
- 의미있는 색상 외에도 텍스트 구분

**연결 화면**: 
- → EXP-ONB-002 (프로필 기본정보)
- ← / (소비자 홈)
- ← /mypage (마이페이지)

**관련 테이블**: `profiles` (읽기 전용)

**관련 RLS 조건**: 
- 사용자가 자신의 프로필만 조회 가능

**Figma 상태**: Draft

**구현 상태**: Not Started

**CTO 결정 필요 사항**: 해당 없음

### EXP-ONB-002: 프로필 기본정보

**화면 ID**: EXP-ONB-002

**화면명**: 프로필 기본정보

**사용자**: 프로필 작성 중인 전문가

**접근 조건**: `isAuthenticated === true` (신규 또는 기존 프로필)

**화면 목적**: 전문가의 기본 정보(이름, 사진, 직군, 한 줄 소개, 상세 소개)를 입력받고 저장한다.

**진입 경로**:
- ← EXP-ONB-001에서 "전문가 프로필 작성 시작" 클릭
- ← EXP-ONB-009에서 편집 링크 클릭
- ← /mypage/profile/edit 직접 접근

**완료 조건**: 모든 필수 정보 입력 + "다음" 클릭 + 저장 성공 → EXP-ONB-003 이동

**핵심 CTA**: `다음` (Primary, 저장 후 이동)

**보조 CTA**: 
- `이전` (이탈 경고 010 포함)
- `프로필 사진 변경` (Secondary)

**표시 데이터**:
- 읽기 전용 필드 (존재 시):
  - 승인 상태 배지
  - 공개 상태 표시
  - 승인일

**입력 데이터**:

**이름 또는 활동명** (필수):
- 유형: 텍스트
- CTO 결정 필요: 글자 수 제한, 특수문자 제한

**프로필 사진** (필수, CTO 결정):
- 유형: 이미지 파일
- 저장소: `profile-images` (private)
- 지원 가능한 파일 형식과 최대 용량은 운영·기술 정책 확정 후 표시합니다
- CTO 결정 필요: 필수 여부, 형식, 크기

**직군** (필수):
- 유형: 선택 (드롭다운)
- 옵션: CTO 결정 필요

**한 줄 소개** (필수):
- 유형: 텍스트
- CTO 결정 필요: 글자 수 제한

**상세 소개** (필수, CTO 결정):
- 유형: 장문 텍스트
- CTO 결정 필요: 글자 수 제한, 필수 여부
- 제약: 의료광고 기준 안내 (자동 검증 아님)

**필수 항목**:
- 이름, 직군, 한 줄 소개 (확정)
- 프로필 사진, 상세 소개 (CTO 결정 필요)

**선택 항목**: 해당 없음

**읽기 전용 항목**: 
- 승인 상태 배지 (존재 시)
- 공개 상태 (존재 시)
- 승인일 (존재 시)

**사용자 변경 금지 항목** (제품 요구사항):
- `verification_status` (사용자 직접 변경 불가)
- `is_public` (사용자 직접 변경 불가)
- `approved_at` (사용자 직접 변경 불가)

**상태별 UI**:
- **001 (로딩)**: "저장 중..." Skeleton 표시
- **002 (빈 상태)**: 신규 작성 시 플레이스홀더 텍스트 (Empty State 분류 안 함)
- **003 (저장 성공)**: "저장되었습니다. 다음 단계로 진행합니다." → 자동 이동
- **004 (저장 실패)**: "저장에 실패했습니다. 다시 시도해주세요." + 재시도 버튼
- **005 (필수 정보 누락)**: 각 필드 옆 "필수 입력" 오류 + 포커스 이동
- **006 (파일 업로드 실패)**: 형식/크기 오류 메시지
- **007 (권한 거부)**: "이 정보를 수정할 수 없습니다."
- **008 (세션 만료)**: "세션이 만료되었습니다."
- **009 (네트워크 오류)**: "저장에 실패했습니다. 다시 시도해주세요."
- **010 (이탈 경고)**: "저장하지 않은 변경사항이 있습니다. 나가시겠습니까?"
- **011 (모바일 키보드)**: CTA와 현재 입력 필드 노출

**오류 처리**:
- 파일 형식 오류: "지원하는 형식 재확인 후 다시 시도해주세요"
- 파일 크기 오류: "파일 크기 재확인 후 다시 시도해주세요"
- 의료광고 표현: 관리자 반려 사유로 처리 (자동 검증 아님)

**빈 상태**: 
- 프로필 사진: "사진 추가" 안내
- 텍스트 필드: 플레이스홀더 텍스트

**모바일 기준**:
- 360px에서 사진 영역 가로 100%
- 파일 선택 버튼 최소 44px
- 입력 필드 최소 16px 패딩
- 텍스트 자연스럽게 줄바꿈

**접근성 기준**:
- 모든 입력 필드에 명확한 라벨
- 포커스 상태 명시
- 오류 아이콘 + 텍스트 (색상만 아님)
- 텍스트 대비 4.5:1 이상

**연결 화면**:
- → EXP-ONB-003 (현재 근무기관)
- ← EXP-ONB-001 (이전)
- ← /mypage (나중에 하기)

**관련 테이블**: `profiles` (⏳ Pending Mapping)

**관련 RLS 조건**:
- 사용자는 자신의 프로필만 수정 가능
- 보호 필드는 수정 불가 (제품 요구사항)
- 기술 구현: M2 종료 후 CTO·기술진 결정

**Figma 상태**: Draft

**구현 상태**: Not Started
