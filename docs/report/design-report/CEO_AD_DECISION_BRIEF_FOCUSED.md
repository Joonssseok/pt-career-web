# CEO AD Decision Brief — 5개 추가 결정 사항

**발신**: PT Career 디자인팀  
**대상**: CEO  
**일자**: 2026-07-21  
**형식**: 각 AD별 1페이지 / 제품정책만 포함  

---

## AD-01: 자격번호 필수 여부

### 결정 사항

```
선택 입력

자격번호를 작성 시 포함할 수 있으나, 
제출 조건이 아님.
```

### 근거

1. **온보딩 진입장벨 감소**  
   자격번호를 몰라도 기관명·취득일만으로 등록 가능 → 초기 완료도 ↑

2. **자격 다양성**  
   민간·국제 자격 중 번호가 없는 것들이 많음

3. **검증은 별도 프로세스**  
   관리팀의 증빙파일 검토 + 검색 시 번호 요청 가능

### 제품정책

```
licenses 테이블:
- license_name (필수)
- issuing_organization (필수)
- acquisition_date (필수)
- license_number (선택)
```

### 기술 선택사항

```
CTO 판정:
- license_number를 별도 검증하거나
- 단순 텍스트 필드로 둘지 결정
```

## AD-02: 증빙파일 허용 형식

### 결정 사항

```
PDF
JPEG
PNG

기타 형식(HEIC, WEBP 등)은 MVP 미지원
```

### 근거

1. **사용성**: 휴대폰 카메라 캡처 또는 스캐너 기본 형식
2. **보안**: 실행파일·압축파일 자동 차단
3. **표준화**: 대부분 브라우저·모바일 기본 지원

### 제품정책

```
업로드 시:
- 파일 형식 확인
- 지원하지 않는 형식은 업로드 거부
- 사용자에게 "PDF, JPG, PNG만 지원" 안내
```

### 운영 안내

```
사용자가 HEIC 또는 기타 형식을 보낼 경우:
"다른 형식입니다. 사진 앱에서 JPEG로 변환 후 
재업로드해주세요" 안내

또는 관리자가 별도 요청
```

### 기술 선택사항

```
CTO 판정:
- MIME 검증 위치 (클라이언트/서버)
- 거부 시 에러 메시지 형식
```

## AD-03: 증빙파일 최대 용량

### 결정 사항

```
파일당 5MB
자격당 최대 1개 파일

예시:
- 자격 3개 등록 → 파일 3개, 각 5MB 이하
```

### 근거

1. **사진 품질**: 5MB로 A4 스캔본이나 휴대폰 사진 충분
2. **업로드 시간**: 일반 네트워크에서 5초 이내
3. **저장소 비용**: MVP 규모 대비 합리적

### 제품정책

```
업로드 시:
- 파일 크기 확인
- 5MB 초과 시 업로드 거부
- 사용자에게 명확한 에러: "파일이 5MB를 초과합니다"
```

### 기술 선택사항

```
CTO 판정:
- 클라이언트 검증 vs 서버 검증
- 오류 처리 방식
```

## AD-04: 센터명·홈페이지 공개범위

### 결정 사항

```
센터명:
사용자가 공개 여부 선택
선택 시 전체 노출

홈페이지:
센터명 공개 시 함께 노출

근무기관 상세주소:
MVP에서는 공개 프로필에 미노출
(내부용으로만 저장)

근무지역:
기존 정책: 사용자 선택적 공개
```

### 근거

1. **신뢰도**: 센터명으로 전문가 신원 확인 가능
2. **개인정보**: 정확한 주소는 추후 단계에서 공개
3. **프로필 품질**: 최소한의 정보로도 신뢰성 구축

### 제품정책

```
공개 프로필에서:
- 센터명: 사용자가 "공개" 선택한 경우만 노출
- 홈페이지: 센터명 공개 시 함께 노출
- 거주지역: 항상 비공개 (본인 전용)
- 근무지역: 사용자 공개 선택 시만 노출
- 상세 주소: 미노출 (관리팀만 조회)

전문가 프로필 (자신의 프로필):
- 모든 정보 조회 가능 (편집 가능)
```

### 기술 선택사항

```
CTO 판정:
- 공개 여부 저장 필드 및 위치
- RLS 정책에 따른 구현
```

## AD-05: 지역 입력 단위와 옵션 체계

### 결정 사항

```
2단계 선택:

1단계 (필수):
시·도 17개
(서울, 부산, 대구, 인천, 광주, 대전, 울산,
 세종, 경기, 강원, 충북, 충남, 전북, 전남,
 경북, 경남, 제주)

2단계 (선택):
시·군·구 (~250개)
예: 서울 선택 후 강남구, 강북구, ... 선택
```

### 적용 대상

```
거주지역:
- 사용자 선택 입력
- 본인 전용 공개
- 1단계 선택 필수, 2단계 선택

근무지역:
- 사용자 선택 입력
- 공개 여부 선택
- 1단계 선택 필수, 2단계 선택
- 근무지역 필터링에 사용
```

### 근거

1. **사용성**: 드롭다운 250개는 사용 불가능 → 2단계 필수
2. **표준화**: 한국의 표준 행정구역 체계 사용 (임의 선별 제외)
3. **필터링**: 지역 단위로 전문가 탐색 가능

### 제품정책

```
지역 드롭다운:
- 1단계: 17개 광역 선택 (필수)
  또는 선택 안 함

- 2단계: 해당 광역의 시군구 선택 (선택)
  또는 선택 안 함

저장: region_id (시군구 수준) 또는 NULL
```

### 데이터 원칙

```
거주지역:
- 선택 입력 (필수 아님)
- 저장 위치: 사용자 프로필 전용
- 근무기관 데이터와 별도 저장
- 본인만 조회 가능

근무지역:
- 선택 입력 (필수 아님)
- 저장 위치: 근무기관 정보와 함께
- 공개 여부를 사용자가 선택
- 공개 시 지역 필터 제공
```

### 마스터 데이터

```
regions 테이블:
- 광역시도 17개 (level 1)
- 시군구 ~250개 (level 2)
- 관계: parent_id로 연결

구성:
- id (PK)
- name
- level (1 or 2)
- parent_id (NULL이면 level 1)
```

### 기술 선택사항

```
CTO 판정:
- regions 테이블 생성 또는 기존 테이블 재사용
- 마스터 데이터 초기 로드 시점
```
