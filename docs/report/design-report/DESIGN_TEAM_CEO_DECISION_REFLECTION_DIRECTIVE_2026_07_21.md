## 1. 작업 목적

CEO가 확정한 Expert Onboarding 제품·운영 결정사항 1~12를 기존 Screen Spec 관련 문서에 반영한다.

이번 작업은 새로운 기능이나 화면을 설계하는 작업이 아니다.

다음 세 문서의 내용을 동일한 정책으로 동기화하는 문서 반영 작업이다.

```
SCREEN_SPEC_EXPERT_ONBOARDING_CTO_REVIEW_V3.md
SCREEN_SPEC_CTO_CEO_DECISION_MATRIX_2026_07_20.md
SCREEN_SPEC_FINAL_REVIEW_REPORT_2026_07_20.md
```

새로운 버전 파일, 마스터 문서 또는 별도의 완료 보고서를 생성하지 않는다.

## 2. CEO 확정사항

### 2.1 이름 또는 활동명

```
최소 2자
최대 40자
```

타인 사칭, 자격 오인 또는 부적절한 표현은 관리자 검토 대상이라는 안내를 유지한다.

---

### 2.2 프로필 사진

```
Draft 작성 중:
선택

검토 요청 시:
필수

승인된 공개 프로필:
필수
```

EXP-ONB-002에서는 사진 없이 임시 저장할 수 있어야 한다.

EXP-ONB-009 또는 EXP-ONB-010에서는 프로필 사진이 없으면 검토 요청을 진행할 수 없도록 표시한다.

사진이 없는 경우 다음 UX를 제공한다.

```
안내:
프로필 검토를 요청하려면 프로필 사진이 필요합니다.

행동:
프로필 기본정보 수정하기
```

---

### 2.3 사진 형식과 용량

```
허용 형식:
JPEG
PNG
WebP

최대 용량:
5MB
```

확장자, MIME type, 압축 및 리사이징 방식은 디자인 문서에서 기술적으로 확정하지 않는다.

해당 항목은 M2 확인 대상으로 연결한다.

---

### 2.4 직군 목록

다음 8개 직군을 반영한다.

```
물리치료사
퍼스널트레이너
건강운동관리사
선수트레이너
필라테스 지도자
요가 지도자
스포츠 코치
기타 운동·재활 전문가
```

다음 원칙을 함께 표시한다.

```
직군 선택은 면허 또는 자격 인증을 의미하지 않는다.

면허와 자격은 별도 정보와 증빙자료 검토를 통해 확인한다.

직군 목록은 향후 CEO 승인에 따라 수정될 수 있다.
```

---

### 2.5 한 줄 소개

한 줄 소개를 선택사항으로 변경한다.

```
필수 여부:
선택

최소 글자 수:
없음

작성 시 최대:
60자
```

한 줄 소개가 없어도 다음 단계로 이동할 수 있고 검토 요청을 진행할 수 있어야 한다.

한 줄 소개 미작성 상태의 공개 프로필 표시 방식은 추후 Figma 단계에서 정의한다.

---

### 2.6 상세 소개

상세 소개를 선택사항으로 변경한다.

```
필수 여부:
선택

최소 글자 수:
없음

작성 시 최대:
500자
```

상세 소개가 없어도 검토 요청을 진행할 수 있어야 한다.

MVP에서는 텍스트 상세 소개만 지원한다.

다음 내용은 Post-MVP 확장 가능성으로만 기록한다.

```
JPEG, PNG, WebP 등 이미지 기반 상세 소개 자료 등록 가능성 검토
```

이번 작업에서 다음 요소를 추가하지 않는다.

- 이미지 상세 소개 업로드 UI
- 이미지 전용 Storage
- 다중 이미지 관리
- PDF 포트폴리오
- 신규 DB 필드

---

### 2.7 거주지역·근무지역 분리

기존의 단일 `지역` 항목을 다음 두 항목으로 분리한다.

#### 거주지역

```
입력:
전문가 선택

조회:
전문가 본인만

관리자 일반 조회:
불가

공개 프로필:
미노출

소비자 검색:
사용하지 않음
```

거주지역은 정확한 주소가 아니라 지역 단위 정보로 표현한다.

구체적인 저장 단위와 기술 구조는 M2 Pending Mapping으로 둔다.

#### 근무지역

```
입력:
전문가 선택

공개 여부:
전문가가 선택

공개 선택 시:
공개 프로필과 검색 결과에 표시 가능

비공개 선택 시:
소비자, 다른 전문가 및 공개 검색 결과에 미노출
```

근무지역과 근무기관 상세 주소를 동일한 정보로 취급하지 않는다.

---

### 2.8 주소 입력 정책

MVP에서는 주소 수동 입력만 제공한다.

```
주소 수동 입력:
포함

주소 검색:
MVP 제외

주소 자동완성:
MVP 제외

지도 SDK:
MVP 제외

지도 마커:
MVP 제외
```

Post-MVP에서는 주소 등록·검색 보조 기능을 검토할 수 있다.

특정 API, MCP 또는 서비스 제공업체를 디자인 문서에서 확정하지 않는다.

## 4. EXP-ONB-003 거주지역·근무지역 분리

기존의 단일 `지역` 항목을 거주지역과 근무지역으로 분리한다.

필수 입력 항목에 두 개 항목을 모두 추가한다.

---

## 5. 데이터 공개 범위 (6가지 분류)

| 분류 | 대표 데이터 | 접근 권한 |
|------|-----------|---------|
| **공개 프로필** | 승인된 공개 기본정보, 경력, 공개 자격·교육·전문분야, 공식 연락처 | 누구나 |
| **선택적 공개** | 근무지역 | 전문가가 공개한 경우 소비자 노출 |
| **소유자 전용** | 거주지역 | 해당 전문가 본인만 |
| **소유자·관리자 제한** | 개인 연락처, 자격번호, 증빙자료 | 해당 전문가와 권한 관리자 |
| **당사자 제한** | 사용자 전달용 반려 사유 | 해당 전문가와 권한 관리자 |
| **관리자 전용** | 내부 관리자 메모 | 권한 관리자만 |

기존 문서에서 거주지역을 관리자 조회 가능 정보로 분류하지 않는다.

---

## 6. 검토 요청 후 수정 정책

상태별 수정 권한을 다음과 같이 반영한다.

```
Draft:
수정 가능

Pending:
수정 불가

Rejected:
수정 가능

Rejected 수정 완료 후:
재제출 가능
```

EXP-ONB-011의 `수정하러 가기` CTA는 제거하거나 비활성 처리한다.

Pending 상태에서는 다음 안내를 표시한다.

```
현재 프로필을 검토하고 있어 수정할 수 없습니다.
검토 결과가 반려된 경우 수정 후 다시 제출할 수 있습니다.
```

Approved 상태의 수정 정책은 이번 범위에서 확정하지 않는다.

---

## 7. 예상 검토 기간

EXP-ONB-010과 EXP-ONB-011에 다음 안내를 반영한다.

```
프로필 검토에는 일반적으로 1~3일이 소요됩니다.
검토 결과는 마이페이지에서 확인할 수 있습니다.
```

1~3일은 예상 기간이며 절대적인 처리기한이나 보장으로 표현하지 않는다.

이메일 알림 기능이 확정되지 않았으므로 이메일 안내를 전제로 한 문구를 사용하지 않는다.

---

## 8. M2 Pending Mapping 항목

다음 10개 항목을 M2 확인 대상으로 명시한다.

```
1. 프로필 실제 DB 필드와 컬럼명
2. 근무기관 데이터 저장 위치
3. 증빙파일 Storage와 접근 정책
4. 연락처 유형 저장 구조
5. 반려 사유와 내부 관리자 메모 저장 구조
6. 거주지역 저장 구조
7. 거주지역 본인 전용 접근 권한
8. 근무지역 저장 구조
9. 근무지역 공개 여부 저장 구조
10. 근무기관 주소와 근무지역의 관계
```

디자인 담당은 실제 필드명, 테이블, RLS 또는 Storage 구조를 정하지 않는다.

## 9. 상태별 화면 수정

### EXP-ONB-002

- 이름과 직군만 작성 단계 필수
- 사진·한 줄 소개·상세 소개는 작성 중 선택
- 사진의 제출 단계 필수 안내 추가

### EXP-ONB-003

- 거주지역과 근무지역 분리
- 근무지역 공개 여부 선택 UI 요구사항 추가
- 주소 검색 CTA 제거
- MVP 주소 수동 입력 기준으로 변경

### EXP-ONB-009

- 프로필 사진 제출 조건 확인
- 거주지역은 본인 전용 정보로 구분
- 근무지역은 공개 여부에 따라 공개 예정 정보를 구분
- 한 줄 소개와 상세 소개 미작성은 누락 오류로 처리하지 않음

### EXP-ONB-010

- 사진 누락 시 제출 차단
- 예상 검토 기간 1~3일 안내
- 한 줄 소개·상세 소개 미작성은 제출 차단 사유에서 제외

### EXP-ONB-011

- Pending 상태 수정 CTA 제거
- 예상 검토 기간 1~3일 안내
- 검토 중 수정 불가 안내

### EXP-ONB-012

- 반려 후 수정 가능
- 수정 완료 후 재제출 흐름 유지
