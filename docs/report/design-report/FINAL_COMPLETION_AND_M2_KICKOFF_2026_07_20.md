## 3. 9번항목 (거주지역·근무지역 분리) — 최우선 관리 항목

### 정의

```
거주지역:
- 입력: 전문가 설정
- 공개: 절대 비공개
- 조회: 전문가 본인만
- DB 상태: Pending Mapping

근무지역:
- 입력: 전문가 설정
- 공개: 전문가 선택
- 공개 시: 공개 프로필 표시
- DB 상태: Pending Mapping
```

### 적용 범위

**EXP-ONB-003 필드 재구성**:
```
기존: 지역 (1개)
현재: 거주지역 + 근무지역 (2개)
```

**데이터 공개 범위 (5가지)**:
```
공개 프로필:
- 근무지역 (선택 시만)

소유자 제한:
- 거주지역 (절대 공개 안 함)

선택적 공개:
- 근무지역 (전문가 선택)

당사자·관리자:
- 반려 사유, 관리자 메모
```

**M2 확인항목 (6~10번)**:
```
6. 거주지역 저장 구조
7. 거주지역 RLS (전문가만 조회·수정)
8. 근무지역 저장 구조
9. 근무지역 공개 여부 저장
10. 근무기관 주소 vs 근무지역 관계
```

### 관리 원칙

✅ **모든 Screen Spec 수정에서 9번항목 명시**
✅ **모든 M2 검증에서 항목 6~10 포함**
✅ **모든 DB 변경에서 9번항목 먼저 구현**
✅ **RLS 정책에서 거주지역 접근 제한 필수**
