```

```

## 실패 시나리오 및 대응

### 시나리오 1: TM 판정 지연

```
상황: CTO 기술 검증 예상 이상 소요

대응:
- Gate 2 지연 시작
- AD 결정 먼저 진행 (Gate 3, 병렬)
- Figma 준비 계속 (실제 설계는 보류)
- 예상 지연: 3-5일
```

### 시나리오 2: CEO 정책 충돌

```
상황: AD 결정이 기술적으로 불가능

대응:
- CTO와 CEO가 협의
- 대체 정책 제시 (예: AD-05 단계 조정)
- Gate 3 재검토
- 예상 지연: 2-3일
```

### 시나리오 3: 개인정보 위험도 RED

```
상황: RLS 정책 구현 어려움 발견

대응:
- Gate 5 FAIL → Gate 6 진행 불가
- CTO·개인정보팀·디자인팀 회의
- 정책 수정 또는 기술 대안 제시
- 예상 영향: M3-A 일정 7~10일 지연
```
