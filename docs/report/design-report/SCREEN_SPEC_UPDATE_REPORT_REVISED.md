**변경 후**: 화면별 적용 매트릭스 (일괄 아님)

각 화면에서 실제로 필요한 상태만 선택적으로 적용:

### 3. 테이블/필드 매핑 검증 표시

**변경 사항**:
- 테이블 구조가 "현재 코드 기반 추정"임을 명시
- **최신 migration 및 DB Schema와의 필드 매핑이 필요**하다고 표시
- 각 화면별 필드 매핑 테이블 추가 (현재 상태, 확인 사항 명시)

### 4. RLS 정책 구현 방식 CTO 결정

**변경 사항**:

**제품 요구사항** (고정):
- `verification_status`, `is_public`, `approved_at`는 사용자가 변경 불가

**구현 방식** (CTO 결정):
- ❌ RLS 정책만으로 구현 가능한가?
- 또는 애플리케이션 로직 + RLS 조합?
- 또는 트리거(Trigger) 자동화?

### 5. 관리자 삭제 권한 제거

**변경 사항**:
- "DELETE: 프로필 삭제는 관리자만" → 제거
- "관리자 프로필 삭제 권한: **CTO 결정 필요**"로 변경

### 6. 개인 연락처 vs 공식 연락처 구분

**변경 사항** (EXP-ONB-003):
- 전화번호 공개 정책 명확화
- "개인 연락처 vs 공식 기관 연락처의 공개 정책을 구분한다"
- **CTO 결정 필요**: 공식 기관 주소/연락처는 공개될 수 있는가?

### 7. 의료광고 검증 → 후속 검토 제안

**변경 사항**:
- 자동 금지 키워드 검증: "확정 기능" → "**후속 검토 제안**"
- 입력 검증이 아닌 관리자 반려 사유로 처리

### 8. M3 구현 정책 명확화
