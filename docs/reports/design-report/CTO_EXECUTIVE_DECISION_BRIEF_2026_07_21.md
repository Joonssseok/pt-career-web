### 2. AD-04·AD-05에 대한 기술 리스크 검토

**AD-04**: 센터명·홈페이지 공개범위 (전문가 선택적 공개)  
**AD-05**: 지역 입력 단위 (시·도 + 시·군·구, 2단계)

**요청**: 
- RLS 정책 복잡도
- 데이터 분산 위험
- 성능 영향도
- 마이그레이션 난이도

---

### 3. Figma 범위 축소 승인

**현재 계획**: 143개 상태 Variant (13개 화면 × 11개 상태)  
**제안 범위**: 약 25~40개 Frame
- 13개 기본 화면
- 실제 필요한 핵심 전체 화면 상태 12~20개
- 공통 오류·로딩·네트워크는 컴포넌트 상태로 처리

**요청**: 
- MVP 범위에서 실제 필요한 상태 제언
- 모바일 QA 최소 기준 조언

---

### 4. M3-A Gate 선결조건 확정

**제안 선결조건**:
- TM-01, 02, 06~10 판정 완료
- AD-04, AD-05 CEO 승인 완료
- Figma Set 1 준비 완료
- 개인정보·RLS 위험도 GREEN

**요청**: CTO 기술 판정 후 위 조건을 M3-A 승인 게이트로 사용할지 여부

---

### 5. M3 구조 확인

**M3-A** (기본 프로필): TM-01, 02, 06~10 필수
