**직군 선택 원칙** (CEO 승인):
- 직군 선택은 면허나 자격 인증을 의미하지 않는다
- 면허와 자격은 자격·면허 정보 및 증빙자료를 통해 별도로 검토한다
- 직군 목록은 향후 CEO 승인에 따라 수정될 수 있다
- 직군 저장 구조는 M2.1에서 현행 구조를 확인하고 CTO 권고안을 작성한다.
  실제 변경은 CEO 기술 변경 승인 후 진행한다.

## IV. 데이터 공개 범위 재분류

**6가지 분류**:

| 분류 | 대표 데이터 | 접근 권한 | 비고 |
|------|-----------|---------|------|
| **공개 프로필** | 승인된 공개 기본정보, 경력, 공개 자격·교육·전문분야, 공식 연락처 | 누구나 | 소비자 공개 프로필에 표시 |
| **선택적 공개** | 근무지역 | 전문가가 공개한 경우 소비자 노출 | 공개 여부를 전문가가 선택 |
| **소유자 전용** | 거주지역 | 해당 전문가 본인만 | 관리자도 일반 조회 불가 |
| **소유자·관리자 제한** | 개인 연락처, 자격번호, 증빙자료 | 해당 전문가와 권한 관리자 | 공개 프로필 미노출 |
| **당사자 제한** | 사용자 전달용 반려 사유 (user_rejection_reason) | 해당 전문가와 권한 관리자 | 내부 관리자 메모 미노출 |
| **관리자 전용** | 내부 관리자 메모 (internal_admin_notes) | 권한 관리자만 | 어떤 사용자에게도 노출 금지 |

**증빙파일 접근**:
- 파일 메타데이터 (파일명, 업로드 상태): 사용자 확인 가능
- 파일 실제 조회·다운로드 방법: M2.1 TM-03에서 현행 Storage·RLS 구조 확인 후 CTO 권고안 작성

## VI. 법률·컴플라이언스 검토 범위

### 의료광고 안내문구 검토

| 항목 | 내용 |
|------|------|
| **담당** | 미지정 |
| **상태** | PENDING / 담당 미지정 |
| **검토 대상** | 의료광고 안내 문구, 허용·금지 표현 |
| **CEO 승인** | 검토 완료 후 필요 |

#### 초안 (사실 기반 경력·활동 설명 권장)

```
프로필 작성 시 다음을 주의하세요:

✓ 작성해도 되는 내용:
  - 실제 경력과 교육, 자격 및 면허
  - 현재 활동 분야와 전문 분야
  - 제공하는 서비스와 수업 유형

✗ 사용하면 안 되는 표현:
  - 치료·완치 또는 특정 결과 보장
  - 비교·최상급·1등 표현 ("최고", "가장 효과적")
  - 환자 후기처럼 보이는 표현
  - 검증되지 않은 전문성 보장
```

#### 법률 검토 담당
```
담당: 미지정

상태: 법률 검토 필요

검토 방식:
필요 시 외부 법률 자문 또는 전문 검토 진행

최종 반영:
CEO 승인 후 확정
```

**주의**: 다음 표현은 문맥에 따라 광고성으로 해석될 수 있으므로 제외
- "경험할 수 있습니다"
- "전문으로 합니다"
- "도와드립니다"
